Tutorial: Variables & JSON Extraction

1 Adding & Setting Global Variables

Global variables are dynamic placeholders you can reference anywhere in your API URLs, Header values, or Body payloads using the double brace format: {{variable_key}}.

  • Defining Variables: Go to the Global Variables tab in the sidebar and click + Add Variable. Provide a unique key name and its initial value.
  • System Environment Variables: You can also reference standard operating system environment variables (e.g., from your host machine or Docker container env). If a template variable is not defined in the database, the backend automatically falls back to searching os.environ. This is ideal for sensitive values like {{API_KEY}} or {{SMTP_PASSWORD}}.
  • Referencing Variables: Inside any text field (URLs, cURL Commands, assertions), type {{ to open the dynamic autocomplete suggestions dropdown listing all available variables.

2 Pre/Post Python Scripts

Complex APIs can run Python before and after the cURL request. Use scripts to prepare dynamic values, fetch OTPs, inspect responses, and persist data back into global variables.

Available API & Global Variables Helpers
  • vars['key'] = 'val' or set_var('key', 'val') saves a global variable (reference as {{key}} in cURLs).
  • vars['key'] or get_var('key') reads a global variable (falls back to env vars).
  • env['key'] = 'val' or set_env('key', 'val') sets an operating system environment variable.
  • env['key'] or get_env('key') reads an operating system environment variable.
  • print("log msg") or log("msg") appends messages to script execution logs displayed in the UI.
  • requests.get(url, headers) makes HTTP calls directly (e.g. to third-party services).
  • ctx['db'].fetch_value(sqlite_url, sql) runs read-only SQLite queries for OTP/test-data lookup.
  • ctx['email'].find_otp(...) searches recent IMAP email and returns the first matching OTP.
  • ctx['response'] is available in post-request scripts with status_code, body, and headers.
Example A: Dynamic OTP Fetching (Pre-Request Script)

Fetches the newest message dynamically from a free Mailsac email inbox using `requests` and saves it as a variable for the main cURL request:

# Get email from variables
email = vars.get("MY_EMAIL", "test-inbox@mailsac.com")

# List messages for the email address
res = requests.get(f"https://mailsac.com/api/addresses/{email}/messages")
messages = res.json()

if messages:
    msg_id = messages[0]["_id"]
    # Retrieve plaintext body of the newest email
    body = requests.get(f"https://mailsac.com/api/text/{email}/{msg_id}").text

Example B: Extracting and Printing ID (Post-Request Script)

Inspects the live response of the current API call and logs the ID of the resource created:

# Access response JSON
body = ctx["response"]["body"]
status = ctx["response"]["status_code"]

if status == 201:
    res_id = body.get("id")
    print(f"Resource created successfully with ID: {res_id}")
    vars["last_created_id"] = res_id
else:
    print(f"Failed to create resource, status: {status}")

3 How to Extract Keys (Dot-Notation JSONPath)

When executing Complex APIs, you can extract values from response JSON payloads and automatically save/overwrite global variables for subsequent runs using dot-notation paths.

Example A: Extracting from an Object
Response JSON 
{
  "status": "success",
  "data": {
    "auth_token": "secret_abc123"
  }
}
Extraction Path: data.auth_token
Result: Saves the value "secret_abc123" to your variable.
Example B: Extracting from an Array
Response JSON 
[
  {
    "id": 105,
    "title": "First Post"
  },
  {
    "id": 208,
    "title": "Second Post"
  }
]
Extraction Path for first element's ID: 0.id (extracts 105)
Extraction Path for second element's Title: 1.title (extracts "Second Post")
Example C: Extracting from Nested Objects & Arrays
Response JSON 
{
  "company": "Crawler Inc",
  "departments": [
    {
      "name": "QA",
      "leads": [
        { "id": 7, "name": "Varun" }
      ]
    }
  ]
}
Extraction Path for lead ID: departments.0.leads.0.id
Result: Saves the value 7.
